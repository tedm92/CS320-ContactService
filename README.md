# ContactService
  Throughout this course, I developed and tested multiple componenets of mobile application, including the contact, task, and appointment services. These projects strenghtened my understanding of software testing, debugging, and secure coding practices
  To ensure that my code and software are both functional and secure, I implemented unit testing using JUnit to validate expected behaviors and identify errors early in development. I focused on input validation, such as enforcing character limits, ensuring required fields were not null, and verifying that phone numbers met proper formatting rules. Additionally, I reviewed my code for potential vulnerabilities and followed secure coding practices to reduce the risk of defects being released into production. Interpreting user needs required translating requirements into functional code. I carefully reviewed the project requirements and ensured that each class and service aligned with what the user expected. For example,I created methods that allowed adding, updating, and deleting objects while maintaining data integrity. I also used testing to confirm that the application behaved as intended from a user perspective.
  When designing software, I approached development by first understanding the requirements and then breaking the system into smaller, manageable components such as objects and services. I followed object-oriented design principles to keep the code organized and maintainable. I also used an iterative approach, continuously testing and refining my code based on feedback and errors encountered during development.
  Overall, these projects enhanced my ability to design, test, and improve software while focusing on both functionality and security. The experience also helped me better understand the importance of thorough testing and validation in real-world software development.
